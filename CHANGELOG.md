# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.0] - 2026-03-31

### Added
- Initial project scaffold generated by Project Generator
- Next.js 15 application with App Router and TypeScript
- Supabase integration for database, auth, and real-time features
- Tailwind CSS and shadcn/ui component library setup
- Complete data model with 10 tables:
  - `users` - User authentication and profiles
  - `businesses` - Business information and settings
  - `business_users` - Many-to-many relationship between users and businesses
  - `platform_integrations` - Connected platform credentials and settings
  - `response_templates` - Customizable response templates by rating level
  - `reviews` - Synced reviews from all connected platforms
  - `review_responses` - Generated and posted responses with status tracking
  - `follow_up_tasks` - Scheduled follow-up actions for negative reviews
  - `analytics_reports` - Performance metrics and insights
  - `audit_logs` - System activity and security logging
- 21 route stubs from site map:
  - `/` - Public landing page
  - `/login` - User authentication
  - `/signup` - User registration
  - `/onboarding` - New user setup flow
  - `/dashboard` - Main dashboard overview
  - `/reviews` - Review management interface
  - `/reviews/:id` - Individual review details
  - `/responses` - Response management
  - `/responses/new` - Create new response
  - `/templates` - Template management
  - `/templates/new` - Create response template
  - `/analytics` - Performance analytics dashboard
  - `/platforms` - Connected platform management
  - `/platforms/connect` - Platform connection flow
  - `/customers` - Customer management
  - `/customers/:id` - Individual customer profile
  - `/team` - Team member management
  - `/settings` - Account and business settings
  - `/api/reviews/sync` - Review synchronization endpoint
  - `/api/responses/generate` - AI response generation
  - `/api/responses/post` - Response posting to platforms
- Integration framework with stubs for:
  - Facebook Business API for review management
  - OpenAI ChatGPT for AI response generation
  - Zapier for workflow automation
  - Gmail for email notifications
  - Zendesk for customer support integration
  - Twilio for SMS notifications
  - HubSpot for CRM integration
- Authentication system with role-based access control:
  - Owner role: Full system access and business management
  - Client role: Business-specific access and settings
  - Reviewer role: Read-only access to reviews and responses
- Project structure following Next.js 15 best practices:
  - App Router with server components by default
  - Separated data access layer in `/db` folder
  - Server actions in `/actions` for form handling
  - Utility functions in `/lib` folder
  - Reusable components with TypeScript strict mode
- Environment configuration for development and production
- Supabase local development setup with migrations
- Vercel deployment configuration
- Complete documentation suite:
  - README.md with setup instructions and project overview
  - CLAUDE.md with AI development guidelines
  - TECHNICAL_DEBT.md tracking known issues
  - ROADMAP.md with feature planning and time estimates
- TypeScript configuration with strict mode enabled
- ESLint and Prettier setup for code consistency
- Git configuration with conventional commit message format

### Infrastructure
- Supabase project configuration with Row Level Security policies
- Database migrations for all core tables
- OAuth configuration for Google My Business integration
- Real-time subscription setup for live review updates
- Error handling and logging framework
- Rate limiting preparation for API endpoints
- Secure credential storage for platform integrations
